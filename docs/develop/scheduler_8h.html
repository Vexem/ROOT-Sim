Function Documentation

void activate_LP ( LID_t  lp,
simtime_t  lvt,
void *  evt,
void *  state 
)

This function is the application-level ProcessEvent() callback entry point. It allows to specify which lp must be scheduled, specifying its lvt, its event to be executed and its simulation state. This provides a general entry point to application-level code, to be used if the LP is in forward execution, in coasting forward or in initialization.

Author
Alessandro Pellegrini
Date
November 11, 2013
Parameters
lpThe id of the LP to be scheduled
lvtThe lvt at which the LP is scheduled
evtA pointer to the event to be processed by the LP
stateThe simulation state to be passed to the LP

void initialize_LP ( LID_t  lp)

This function initializes a LP execution context. It allocates page-aligned memory for efficiency reasons, and then calls <context_create>() which does the final trick. <context_create>() uses global variables: LPs must therefore be intialized before creating new kernel threads for supporting concurrent execution of LPs.

Author
Alessandro Pellegrini
Date
November 8, 2013
Parameters
lpthe idex of the LP in the LPs descriptor table to be initialized
